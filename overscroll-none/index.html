overscroll-behavior: none. Sets overscroll-behavior: none on html and body to disable rubber-band scrolling. If the canvas fill is triggered by the elastic overscroll effect at the top of the page, disabling it should prevent the fill — even at scrollY=0. No scroll guard needed.
html, body { overscroll-behavior: none; } /* + root excluded from VT (like V33) */ :root, html, body, .page { view-transition-name: none !important; }
View Transition Card

No scroll guard needed. With overscroll-behavior disabled, Safari shouldn't enter the elastic overscroll state at the top of the page. If the canvas fill is triggered by that overscroll state, disabling it should fix the top-of-page edge case.

Note: this also disables pull-to-refresh on Safari. Acceptable tradeoff if it fixes the bug.
