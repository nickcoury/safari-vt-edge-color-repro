Variant 31
Toggle Fixed/Sticky to Absolute During VT. Before the VT callback, a class is added to body that converts all position: fixed and position: sticky elements to absolute. Removed in .finished.finally(). If any fixed/sticky elements in the viewport are the tint source, this should neutralize them.
// JS: before startViewTransition document.body.classList.add('vt-running'); // Force all fixed/sticky to absolute document.querySelectorAll('*').forEach(el => { const pos = getComputedStyle(el).position; if (pos === 'fixed' || pos === 'sticky') { el.dataset.vtOrigPos = pos; el.style.position = 'absolute'; } }); // After transition.finished.finally(): // restore original positions
View Transition Card

During the VT callback, any element with position: fixed or sticky is temporarily converted to absolute. This prevents them from being anchored to the viewport edges during the snapshot.

If the safe area tint comes from a fixed-position element (like the VT overlay itself or a sticky header), this should neutralize it.
