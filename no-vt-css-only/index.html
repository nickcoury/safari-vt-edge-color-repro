CSS-Only Card

This card uses zero View Transitions API. The expand/collapse is a plain CSS transition: max-height 0.4s. No view-transition-name, no startViewTransition(), no pseudo-elements, no canvas fill.

Safe areas should remain completely transparent throughout — just like they are when you scroll normally. The tradeoff: no shared-element morph, just a smooth height expansion.

No VT = No Bug

Since there's no View Transition, Safari never creates the VT overlay or fills the canvas. Safe areas behave exactly like normal scrolling.

Tradeoff

You lose the shared-element crossfade/morph effect. But you gain reliable safe-area transparency. If the VT API is the only thing causing the bug, this is the workaround.
