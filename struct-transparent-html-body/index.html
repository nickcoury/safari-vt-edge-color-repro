Variant 15
Structural: Transparent HTML/Body. html and body have background: transparent always. The white background lives only on the .page wrapper div, which is inset with safe-area padding. The regions outside .page (including safe areas) are genuinely transparent — not white.
html { background: transparent; } body { background: transparent; } .page { background: white; }
View Transition Card

With html and body transparent, the safe area regions outside the content wrapper have no white background. When Safari captures the viewport snapshot for the VT, those regions should come through as transparent rather than white.

If Safari still fills them with white, it means Safari is actively filling the snapshot with a default background color regardless of the actual page background.
