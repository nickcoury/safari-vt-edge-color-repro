Transparent Fixed Strips. Two always-present position: fixed strips at top:0 and bottom:0 with background: transparent, full width, safe-area height (min 6px), pointer-events: none. Gives Safari's sampler a transparent fixed element to lock onto in the safe-area bands instead of falling back to the page/canvas color.
.safe-strip-top { position: fixed; top: 0; left: 0; width: 100%; height: max(env(safe-area-inset-top), 6px); background: transparent; z-index: 9999; pointer-events: none; } .safe-strip-bottom { position: fixed; bottom: 0; left: 0; width: 100%; height: max(env(safe-area-inset-bottom), 6px); background: transparent; z-index: 9999; pointer-events: none; }
View Transition Card

There are invisible fixed strips at the very top and bottom of the viewport. They're transparent and don't block touch. When Safari samples the safe-area bands during VT, it should find these fixed elements with background: transparent instead of falling back to the canvas white.
