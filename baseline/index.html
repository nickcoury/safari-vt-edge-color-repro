View Transition Card

This card uses view-transition-name: card — a shared element. Tap anywhere to trigger the transition. On Safari with edge-to-edge displays, the root view transition snapshot paints the safe areas as solid white.

The View Transitions API captures the entire viewport as a static snapshot for the root crossfade. On notched iPhones using viewport-fit=cover, this snapshot flattens the transparent header/footer regions into the page's background color (white), creating a visible flash.

Section One — Setup

This content forces the page well beyond a single viewport. On a typical phone display (~800px), this page spans roughly 2.5 viewports.

The white background should remain solid through the entire transition — no black flashes, no bleed from underlying layers.

Section Two — Safari View Transitions

Safari shipped support for same-document View Transitions in Safari 18.2. Shared element transitions should smoothly animate the card between states.

On notched iPhones, the view transition overlay layer may not respect the page background behind the status bar, resulting in a flash of a different color.

Section Three — Expected vs. Actual

Expected: Smooth transition with no color flashes in safe areas.

Actual (bug): The Dynamic Island / notch area briefly renders solid white instead of transparent during the transition.

Section Four — Environment

Test on Safari iOS 18.2+ on a device with notch or Dynamic Island. iPhone 14 Pro or later. Most visible in light mode with white background.
