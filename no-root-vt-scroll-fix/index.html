Variant 35
No Root VT + Scroll Guard. Same as V33 (root excluded from VT) PLUS a JS guard: if scrollY === 0 when VT is triggered, the page is scrolled 1px down first. This prevents the Safari top-of-page canvas fill edge case. After VT, scroll position is restored.
// Before startViewTransition: const wasAtTop = scrollY === 0; if (wasAtTop) scrollTo(0, 1); // 1px nudge document.startViewTransition(() => updateCardState()); // After transition.finished: if (wasAtTop) scrollTo(0, 0); // restore
View Transition Card

V26/V33 work except at scrollY=0. This variant adds a 1px scroll guard: if you're at the very top of the page when you tap, the page nudges down 1px before starting the VT, then restores after.

The 1px scroll is imperceptible but prevents Safari's top-of-page canvas fill. If this works consistently at ALL scroll positions (including top), we have a reliable workaround.
