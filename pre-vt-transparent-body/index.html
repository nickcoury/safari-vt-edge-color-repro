Variant 25
Pre-VT Transparent Body. Sets body to transparent before calling startViewTransition(), not inside the callback. This means the OLD snapshot (taken before the callback) captures transparent body too. Restored after transition.finished. Unlike V16 which only affected the NEW snapshot.
// BEFORE startViewTransition: document.body.style.background = 'transparent'; const t = document.startViewTransition(() => { updateCardState(); // body still transparent for NEW snapshot too }); t.finished.then(() => { document.body.style.background = 'white'; });
View Transition Card

Key difference from V16: V16 set transparent inside the callback, which only affects the NEW snapshot (taken after callback). This variant sets transparent BEFORE startViewTransition, so the OLD snapshot also captures a transparent body.

If both snapshots have transparent body, and Safari still shows white, then transparent is definitively resolved to white at the compositor level — not a timing issue.
